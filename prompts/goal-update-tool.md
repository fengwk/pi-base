Update the current goal to `complete` or `blocked`. Every update requires a non-empty `reason` with detailed rationale and concrete evidence: for `complete`, identify the satisfied requirements and supporting evidence; for `blocked`, identify the repeated blocker, attempts made, and required user input or external change. Set `complete` only when current evidence proves every objective requirement is satisfied and no required work remains. Set `blocked` only after the same blocking condition has repeated for at least three consecutive goal turns and meaningful progress cannot continue without user input or an external-state change. Do not use this tool to pause, resume, clear, or budget-limit a goal.
